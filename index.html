Entity Resolution · NLP · Semantic Search
Product Matching Pipeline

Solves the core e-commerce problem: millions of sellers list the same product with different titles, abbreviations, and formats. The system identifies which listings refer to the same real-world product.

The problem with vanilla transformers: "Sony XM5 Headphones" vs "Sony XM4 Headphones" scores 0.97 cosine similarity — nearly identical, but they're different products. A single digit difference gets drowned out by surrounding shared words.

Two-stage solution: Sentence Transformer (all-MiniLM-L6-v2) for semantic similarity, followed by an Attribute Guard that detects model code, storage, and size conflicts using regex. Conflicts multiply the score by a penalty (0.40–0.50), collapsing the false match from 0.97 → 0.49.

Text Classification · DistilBERT · Multi-task Learning
Hierarchical Taxonomy Classifier

Automates product categorization for e-commerce listings. Given any product title, predicts the full 3-level category path (e.g., Electronics → Audio → Headphones) in under 100ms on CPU.

Architecture: Fine-tuned DistilBERT with 3 simultaneous classification heads (5 / 12 / 18 classes) trained end-to-end. One shared encoder learns features for broad and fine-grained taxonomy simultaneously — the combined loss (L1 + L2 + L3) backpropagates through all three heads in a single step, forcing the encoder to learn representations useful at every level of the hierarchy.

Why DistilBERT over BERT? 40% smaller, 60% faster, retains 97% performance — runs in ~80ms vs ~200ms per title on CPU, critical for listing ingestion pipelines.
100% L1 accuracy 81% L3 accuracy 73% on unseen product types

Named Entity Recognition · BIO Tagging · Token Classification
Product Attribute Extractor (NER)

Turns raw, unstructured product titles into structured, queryable attributes. Feeds directly into the Entity Resolution pipeline — the Attribute Guard uses these extracted values for conflict detection.

9 entity types: BRAND, MODEL, COLOR, STORAGE, SIZE, MATERIAL, NETWORK, GENERATION, COUNT. Handles multi-word entities like "Le Creuset", "Deep Purple", "Cast Iron" using BIO tagging (B = beginning of span, I = inside span, O = not an entity).

Subword alignment: DistilBERT splits "iPhone" into ["i", "##phone"]. Labels are predicted only for the first subword of each word; continuation subwords are masked with -100 in the loss — the industry standard for BERT-based NER. Evaluated with seqeval at the span level, not token level.

Computer Vision · EfficientNet · Explainable AI
Product Image Quality Scorer

Scores product photos on a 1–5 scale and generates actionable seller feedback — not just a number, but exactly what is wrong and how to fix it. Built for eBay-style listing quality enforcement.

Two-stage architecture: EfficientNet-B0 (4.3M params, ImageNet pretrained, MSE regression head) combined with 7 classical CV signals — Laplacian sharpness, brightness penalty curve, pixel std contrast, Hasler–Süsstrunk colorfulness, Canny edge density (background simplicity), minimum resolution, Gaussian residual noise. Final score = 0.65 × CV score + 0.35 × DL score.

Why combine both? CV signals are fast, explainable, and reliable for specific measurable properties — when a seller scores poorly, we can tell them exactly why ("Laplacian variance: 0.03 — your image is blurry"). EfficientNet captures holistic quality that's hard to measure explicitly. Diagnosis always comes from CV signals, never from the neural network, ensuring advice is always grounded.
7 CV signals Correct ranking on all test images Actionable feedback per image

Computer Vision · Anomaly Detection · Manufacturing
Manufacturing Defect Detection System

A real-time quality control pipeline for electronics manufacturing — flags anomalous products during inspection before they reach downstream assembly or customers.

Architecture: Convolutional autoencoder trained exclusively on defect-free (normal) images. At inference time, any image with high reconstruction error is flagged as anomalous — the model learns what "normal" looks like and rejects everything that deviates. Threshold is calibrated on a validation set to balance precision and recall. Optionally extended with a supervised ResNet-18 classifier for class-level defect categorization (scratch, contamination, bent component) when labeled defect images are available.

Why unsupervised? Labeled defect data is expensive to collect in manufacturing. Training only on good images means the system can be deployed with zero defect labels — it learns the distribution of normal products and flags anything outside it. Evaluated with AUROC and F1, with tunable reconstruction error thresholds.
MVTec AD dataset Up to 40% inspection labour reduction Real-time Streamlit dashboard
Industry benchmarks: early anomaly detection systems of this type reduce manual inspection labour by up to 40% and catch defect rates human inspectors miss at high throughput.

LLM Observability · Full-stack · Production ML
AI Guardian

The hardest part of deploying a GenAI system is not building it — it's knowing whether it still works correctly three weeks later. AI Guardian solves this with event-driven LLM monitoring in production.

How it works: Python SDK (@track_llm_call decorator) captures latency, token usage, cost, quality scores, and hallucination risk for every LLM call without modifying application code. Threshold-based alerting fires before issues escalate. Multi-step trace tracking follows RAG pipeline chains end-to-end. All metrics flow into a PostgreSQL backend and surface in a React dashboard with live WebSocket updates — so engineering and business teams can monitor model behaviour without querying the database directly.

Works with any LLM provider: OpenAI, Anthropic, Google. Designed to be dropped into an existing LLM application with a single decorator.

Deep Learning · CNN · Low-Resource NLP
Arabic Handwriting Recognition

CNN trained from scratch to classify all 28 Arabic handwritten characters across 13,440 labeled images. Goal: verify that a well-designed custom CNN can compete with transfer learning on a low-resource task.

Design decisions: Data augmentation (rotation, shear, zoom) applied to address class imbalance. Dropout regularization to prevent overfitting on the 13K-image dataset. BatchNorm for training stability. Confusion matrix analysis identified the 3 character pairs with the highest confusion rates — ب/ت/ث (similar baseline strokes) and و/ر (similar curved shapes) — which informed post-training threshold adjustment.

Why train from scratch? Transfer learning from ImageNet is not ideal for Arabic handwriting — stroke patterns and character anatomy don't map well to natural image features. A CNN designed specifically for the structure of Arabic script outperforms naive transfer.
92.4% test accuracy 28 character classes 13,440 labeled images
